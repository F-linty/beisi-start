### 整体架构

* 前端部分：Vue3 + pinia(持久化) + Element-Plus
* 后端部分：Nest.js(基座使用的是：Fastify.js)

### 细节说明

> 前端部分：
* 账号密码进行非空验证。
* 登入后pinia的持久化在浏览器环境会把，token和refToken存入localStorage中。
* 请求个人信息时，会携带token，后端验证token，返回用户信息。
* 预留了`handleLoginSuccess`函数用来处理登录之后的逻辑，可以进一步设计
> 后端部分
* 设计了全局的异常处理和路由守卫
* 异常处理用来处理各种处理错误，认证错误，返回逻辑里面符合接口设计目标的部分。
* 验证用户身份时依旧会验证前端发来的账号密码是否为空返回提示
* 设计了双Token模式，登录验证通过之后将返回token和refToken。前端可以在下一请求中携带

### 接口说明
* 接口描述

  用户登录接口，验证用户名和密码，返回登录状态及模拟 Token。

* 请求接口

  - URL：users/login
  - 请求方式：POST
  - 请求参数类型：application/json

* 请求参数

  | 参数名   | 类型   | 是否必填 | 说明   |
  | -------- | ------ | -------- | ------ |
  | account | string | 是       | 用户名 |
  | password | string | 是       | 密码   |

* 请求示例

  ```json
  {
    "account": "ycx123",
    "password": "123456"
  }
  ```

* 响应结果

  * 成功响应（账号密码正确）

    * HTTP 状态码：200

      ```json
      {
        "code": 200,
        "message": "登入成功",
        "data": [{
          "token": "abcdef123456",
          "refToken": "abcdef123456"
        }]
      }
      ```

    * 字段说明

      | 字段名     | 类型   | 说明                   |
      | ---------- | ------ | ---------------------- |
      | code       | int    | 200 表示成功             |
      | message    | string | 提示信息               |
      | data[0] | Array | 登录成功后的token |

  * 失败响应（账号密码错误）

    * HTTP 状态码：404

      ```json
      {
        "code": 404,
        "message": "用户不存在",
        "data": []
      }
      ```

    * 字段说明

      | 字段名  | 类型   | 说明               |
      | ------- | ------ | ------------------ |
      | code    | int    | 404 表示找不到用户         |
      | message | string | 错误提示信息       |
      | data    | Array   | 登录失败返回空数组 |

      * HTTP 状态码：401

      ```json
      {
        "code": 401,
        "message": "密码错误",
        "data": []
      }
      ```

    * 字段说明

      | 字段名  | 类型   | 说明               |
      | ------- | ------ | ------------------ |
      | code    | int    | 401 表示认证失败         |
      | message | string | 错误提示信息       |
      | data    | Array   | 登录失败返回空数组 |

* 接口描述

  用户信息获取接口，验证token，返回用户信息。

* 请求接口

  - URL：/users
  - 请求方式：GET
  - 认证参数：Authorization:`Bearer ${token}`
* 请求参数

  | 参数名   | 值格式 | 说明   |
  | -------- | ------| ------ |
  | Authorization | Bearer token | 用户token |

* 请求示例

  ```json
  {
    "Authorization": `Bearer ${token}`
  }
  ```

* 响应结果

  * 成功响应（账号密码正确）

    * HTTP 状态码：200

      ```json
      {
        "code": 200,
        "message": "查询成功",
        "data": [{
            userName:"小杨",
            account:"ycx123",
            avatar:"https://www.keaitupian.cn/cjpic/frombd/1/253/3663778712/1545220977.jpg"
        }]
      }
      ```

    * 字段说明

      | 字段名     | 类型   | 说明                   |
      | ---------- | ------ | ---------------------- |
      | code       | int    | 200 表示成功             |
      | message    | string | 提示信息               |
      | data[0] | Array | 用户信息 |

  * 失败响应（账号密码错误）

    * HTTP 状态码：404

      ```json
      {
        "code": 404,
        "message": "用户不存在",
        "data": []
      }
      ```

    * 字段说明

      | 字段名  | 类型   | 说明               |
